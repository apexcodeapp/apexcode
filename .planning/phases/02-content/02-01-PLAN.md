### Design Token Reference (from global.css and UI-SPEC)

All Tailwind utility class names map to CSS custom properties already declared in `src/styles/global.css`. Use the token names below — never raw hex values.

| Token | Utility class | Value |
|-------|--------------|-------|
| Page background | `bg-surface` | `#0a0a0a` |
| Card / header bg | `bg-surface-raised` | `#111111` |
| Primary text | `text-text-primary` | `#f5f5f5` |
| Muted text | `text-text-muted` | `#a1a1a1` |
| Border | `border-border` | `#1f1f1f` |
| Accent (CTA bg) | `bg-accent` | `#ffffff` |
| Surface (CTA text) | `text-surface` | `#0a0a0a` |
| Decorative number | `text-border` | `#1f1f1f` |

### Typography Reference (from UI-SPEC)

| Role | Mobile | Desktop | Weight | Tailwind |
|------|--------|---------|--------|---------|
| Display (h1) | 32px | 48px | 600 | `text-[32px] md:text-[48px] font-semibold leading-none tracking-[-0.02em]` |
| Heading (h2) | 22px | 28px | 600 | `text-[22px] md:text-[28px] font-semibold leading-tight tracking-[-0.01em]` |
| Card title (h3) | 16px | 16px | 600 | `text-base font-semibold` |
| Body | 16px | 16px | 400 | `text-base font-normal leading-relaxed` |
| Label / caption | 14px | 14px | 400 | `text-sm font-normal leading-relaxed` |

### Section Shell Pattern

Every section component follows this structure:

```astro
---
// empty or import-only frontmatter
---

<section id="{id}" aria-labelledby="{id}-heading" class="py-16 md:py-24">
  <div class="max-w-5xl mx-auto px-4">
    <h2 id="{id}-heading" class="text-[22px] md:text-[28px] font-semibold leading-tight tracking-[-0.01em] text-text-primary mb-12">
      Section Title
    </h2>
    <!-- content -->
  </div>
</section>
```

Exception: Hero uses `aria-label="Hero"` (not `aria-labelledby`) because binding the landmark label to the `<h1>` causes some screen readers to announce the h1 twice.

Implementation notes:
- Three steps per PROC-01: Discover → Build → Ship
- Each step has a title and one-sentence description per PROC-02
- Number spans (`01`, `02`, `03`) are hardcoded strings — not CSS counters — per research recommendation. Simpler, more maintainable, eliminates CSS counter edge cases
- `aria-hidden="true"` on each number span — decorative only; screen readers encounter the `<h3>` titles directly
- `text-border` on number spans = `#1f1f1f` — intentionally low contrast (~1.2:1 on #0a0a0a background) making numbers a design element without competing with headings. Accessibility is preserved because `aria-hidden="true"` removes them from the a11y tree
- `select-none` prevents accidental text selection of large decorative numbers
- Step titles use `<h3>` to maintain heading hierarchy: h1 (hero) → h2 (section) → h3 (step)
- `grid-cols-1 md:grid-cols-3 gap-8` — wider gap than service cards (gap-8 vs gap-6) to breathe more around the large decorative numbers
