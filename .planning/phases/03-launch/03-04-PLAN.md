  <read_first>
    - .planning/phases/03-launch/03-RESEARCH.md (section: "Sitemap Submission to Google Search Console")
    - public/robots.txt (confirm the Sitemap URL matches what will be submitted)
  </read_first>
  <action>
    **Step 1: Add property**
    1. Go to https://search.google.com/search-console/
    2. Click "Add property"
    3. Choose "URL prefix" type and enter your production URL (e.g., `https://apexcode.dev/`)
    4. Alternatively, choose "Domain" type for `apexcode.dev` (covers all subdomains)

    **Step 2: Verify ownership**
    - Preferred: DNS TXT record verification
      1. Copy the TXT record value from GSC
      2. Add it to your domain's DNS settings at your registrar
      3. Wait for propagation (can take minutes to hours)
      4. Click "Verify" in GSC
    - Alternative: HTML file verification
      1. Download the HTML verification file from GSC
      2. Place it in the `public/` directory of the project
      3. Commit + push so it deploys
      4. Click "Verify" in GSC

    **Step 3: Submit sitemap**
    1. In GSC left nav: Indexing > Sitemaps
    2. Enter: `sitemap-index.xml` (NOT `sitemap-0.xml`)
    3. Click Submit
    4. Status should show "Success" or "Pending" (Google takes time to process)

    **Step 4: Check index coverage**
    1. In GSC left nav: Indexing > Pages
    2. Verify no errors reported
    3. Note: It may take 24-48 hours for Google to crawl and index the site

    **Important:** Submit `sitemap-index.xml`, not `sitemap-0.xml`. The index file references all numbered sitemaps and is the correct entry point per Google's guidelines.
  </action>
  <acceptance_criteria>
    - GSC shows the property as "Verified" (green checkmark)
    - Sitemaps section shows `sitemap-index.xml` with status "Success" or "Pending"
    - Pages/Index coverage section shows no errors (may show "Discovered" or "Crawled" status for the homepage)
  </acceptance_criteria>

  </verify>
