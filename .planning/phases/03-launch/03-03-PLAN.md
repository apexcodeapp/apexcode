<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Deploy site to production</name>
  <read_first>

  </read_first>
  <files>none</files>
  <action>
    Connect the GitHub repo to the chosen hosting platform and trigger the first deploy.

    **If Cloudflare Pages:**
    1. Go to https://dash.cloudflare.com/ > Workers and Pages > Create > Pages > Connect to Git
    2. Select repository: `apexcodeapp/apexcode`, branch: `master`
    3. Framework preset: select "Astro"
    4. Build command: `npm run build`
    5. Build output directory: `dist`
    6. Go to Settings > Environment Variables > add: `NODE_VERSION` = `22`
    7. Trigger deploy (or it auto-deploys on connect)
    8. Note the assigned URL (e.g., `apexcode.pages.dev`)
    9. If custom domain: Settings > Custom domains > add your domain > follow DNS instructions

    **If Vercel Pro:**
    1. Go to https://vercel.com/dashboard > upgrade to Pro plan
    2. Import repo: `apexcodeapp/apexcode`
    3. Framework: Astro (auto-detected)
    4. Deploy (zero-config)
    5. Note the assigned URL
    6. If custom domain: Settings > Domains > add your domain

    **After deploy succeeds:** Confirm the site loads at the production URL. Verify the URL matches what was configured in Plan 01 (astro.config.mts site value).
  </action>
