    ```astro
        </div>
      </div>
    </section>
    ```

    Key changes: `background-image` CSS removed. `<img>` with `fetchpriority="high"` and `loading="eager"` added as a positioned element behind the text content. `overflow-hidden` on the parent to contain the image within the rounded corners. `relative z-10` on the text wrapper to layer above the image.

    **2. Layout.astro — Add hero image preload hint:**

    Add this `<link>` tag inside `<head>`, after the favicon link and before the `<SEO>` component:

    ```html
    <link rel="preload" href="/hero-bg.jpg" as="image" fetchpriority="high" />
    ```

    This tells the browser to start fetching the hero image during HTML parsing, before the image element is discovered in the DOM.

