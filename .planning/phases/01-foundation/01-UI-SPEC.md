## Spacing Scale

Declared values (multiples of 4 only). Tailwind v4 token names map to CSS custom properties in `@theme {}`.

| Token | Value | Tailwind v4 variable | Usage |
|-------|-------|----------------------|-------|
| xs | 4px | `--spacing-1` | Icon gaps, inline padding, border-radius offsets |
| sm | 8px | `--spacing-2` | Compact element spacing, tag padding |
| md | 16px | `--spacing-4` | Default element spacing, input padding, button padding |
| lg | 24px | `--spacing-6` | Card internal padding, nav item gaps |
| xl | 32px | `--spacing-8` | Section inner gaps, component separation |
| 2xl | 48px | `--spacing-12` | Major section breaks within a page section |
| 3xl | 64px | `--spacing-16` | Page-level section vertical padding (top/bottom) |
| 4xl | 96px | `--spacing-24` | Hero section top padding on desktop |

Exceptions:
- Sticky header height: 56px (not on the 8-point scale — required to clear anchor-scroll offsets; use `scroll-margin-top: 56px` on all section IDs). Source: NAV-01 requirement.
- Mobile touch targets (links, nav items): minimum 44px hit area even if visual height is smaller. Apply via `min-h-[44px]` and `flex items-center`.

**Source:** FOUND-04 (mobile-first), NAV-01 (sticky header), default 8-point scale convention

## Typography

Font: **Geist Sans** — a modern, high-legibility variable font designed by Vercel. Matches the design reference exactly. Self-hosted via `@fontsource/geist-sans` to eliminate the render-blocking Google Fonts external request.

Weights used: `400` (regular) and `600` (semibold). No other weights. Tailwind v4 maps these via `font-normal` and `font-semibold`.

| Role | Size | Weight | Line Height | Tailwind class | Usage |
|------|------|--------|-------------|----------------|-------|
| Body | 16px | 400 | 1.6 | `text-base font-normal leading-relaxed` | Paragraph text, nav links, footer copy |
| Label | 14px | 400 | 1.4 | `text-sm font-normal leading-snug` | Caption text, metadata, tag labels |
| Heading | 28px | 600 | 1.25 | `text-[28px] font-semibold leading-tight` | Section titles (`<h2>`) |
| Display | 48px | 600 | 1.1 | `text-[48px] font-semibold leading-none` | Page-level hero title (`<h1>`) — desktop |

Mobile scaling:
- Display: 32px on mobile (`text-[32px] md:text-[48px]`). Source: FOUND-04 mobile-first.
- Heading: 22px on mobile (`text-[22px] md:text-[28px]`).
- Body and Label: no scaling — 16px and 14px work at all viewport widths.

Letter spacing:
- Display: `-0.02em` (tight tracking reinforces Vercel aesthetic)
- Heading: `-0.01em`
- Body and Label: `0` (default)

**Source:** PROJECT.md (Vercel design reference), RESEARCH.md (Vercel-like aesthetic), FOUND-06 (high contrast), FOUND-04 (mobile-first). Type scale default applied per Vercel.com reference.

## Color

All colors declared as CSS custom properties inside `@theme {}` in `src/styles/global.css`. Tailwind v4 maps them automatically.

| Role | Value | Tailwind token | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | `#0a0a0a` | `--color-surface` | Page background, all surfaces |
| Secondary (30%) | `#111111` | `--color-surface-raised` | Header background (sticky), card backgrounds, footer background |
| Accent (10%) | `#f5f5f5` | `--color-text-primary` | Primary text — body copy, headings, nav links |
| Text muted | `#a1a1a1` | `--color-text-muted` | Secondary text — label, caption, footer fine print |
| Border | `#1f1f1f` | `--color-border` | Card borders, header bottom border, dividers |
| Accent interactive | `#ffffff` | `--color-accent` | Specific interactive elements only (see below) |
| Destructive | not applicable | — | No destructive actions in Phase 1 |

**Accent reserved for:**
- Primary CTA button background (Phase 2)
- Anchor link hover underline on nav items
- Active nav indicator (sticky header current-section highlight)

The 60/30/10 breakdown maps to: near-black background everywhere (60%), slightly lighter header/footer/card surfaces (30%), and off-white text plus the single interactive white accent (10%).

**Source:** REQUIREMENTS.md FOUND-06 (`#0a0a0a`, not pure `#000000`), PROJECT.md (Vercel design reference, high contrast), RESEARCH.md pitfall 4 (pure black accessibility — use near-black `#0a0a0a`, off-white `#f5f5f5` not `#ffffff` for body text)

## Component Inventory (Phase 1 scope)

These are the components that Phase 1 must produce. Each is a plain `.astro` file — no React, no client-side JS.

| Component | File path | What it renders |
|-----------|-----------|-----------------|
| Layout shell | `src/layouts/Layout.astro` | `<html>`, `<head>` with SEO meta via `astro-seo`, global styles import, `<slot />` |
| SiteHeader | `src/components/layout/SiteHeader.astro` | Sticky `<header>`: logo/wordmark left, anchor nav links right (max 4 items) |
| SiteFooter | `src/components/layout/SiteFooter.astro` | `<footer>`: copyright line, studio name, email link |
| Page wrapper | `src/pages/index.astro` | Minimal shell — Layout + SiteHeader + SiteFooter + empty `<main>` |
| Metadata constants | `src/lib/metadata.ts` | `SITE_URL`, `SITE_TITLE`, `SITE_DESCRIPTION` — single source of truth for SEO |
| Global CSS | `src/styles/global.css` | `@import "tailwindcss"`, `@theme {}` blocks, base reset |

**SiteHeader specification:**
- Position: `sticky top-0 z-50`
- Height: 56px (`h-14`)
- Background: `#111111` (secondary surface) with a 1px bottom border at `#1f1f1f`
- Logo: text wordmark "Apex Code" in Geist Sans 16px weight 600, color `#f5f5f5`
- Nav links: 4 items maximum — Services, Why Us, Process, Contact — each is an `<a href="#section-id">` anchor link
- Nav link styles: 14px, weight 400, color `#a1a1a1` default; `#f5f5f5` on hover; transition 150ms ease
- Mobile: nav links collapse to hidden on viewports below `md` breakpoint (768px). No hamburger menu in Phase 1 — mobile visitors reach sections by scrolling. Phase 2 can add a mobile menu if needed.
- Backdrop: `backdrop-blur-sm` for subtle depth when content scrolls under it

**SiteFooter specification:**
- Background: `#111111` (secondary surface)
- Top border: 1px `#1f1f1f`
- Padding: 32px vertical
- Content: one line — "© 2026 Apex Code. All rights reserved." — 14px, weight 400, color `#a1a1a1`

**Source:** REQUIREMENTS.md NAV-01 (sticky header, anchor links), ROADMAP.md Phase 1 goal (layout shell), RESEARCH.md architecture patterns

## Interaction Contract

Phase 1 contains exactly one class of interaction: anchor-link navigation in the sticky header.

| Interaction | Behavior |
|-------------|----------|
| Nav link click | Smooth scroll to section via CSS `scroll-behavior: smooth` on `<html>` — no JS needed |
| Nav link hover | Color transition from `#a1a1a1` to `#f5f5f5` over 150ms ease |
| Header scroll behavior | Header stays fixed at top (`sticky top-0`) — content scrolls beneath it |
| Scroll offset for anchors | All section `id` elements use `scroll-margin-top: 56px` to clear the sticky header height |

No JavaScript interactions in Phase 1. No modals, no toggles, no animations. Source: FOUND-05 (lean JS), RESEARCH.md (zero-JS Astro output).

## Accessibility Baseline

| Requirement | Implementation |
|-------------|----------------|
| Color contrast (text) | `#f5f5f5` on `#0a0a0a` — contrast ratio 18.9:1 (exceeds WCAG AAA 7:1) |
| Color contrast (muted text) | `#a1a1a1` on `#0a0a0a` — contrast ratio 5.7:1 (passes WCAG AA 4.5:1) |
| Color contrast (muted on secondary) | `#a1a1a1` on `#111111` — contrast ratio 5.5:1 (passes WCAG AA) |
| Focus indicators | Default browser focus ring — do not remove `outline: none` without a custom replacement |
| Skip link | `<a href="#main-content" class="sr-only focus:not-sr-only">Skip to main content</a>` — first focusable element in `<body>` |
| Semantic HTML | `<header>`, `<nav>`, `<main>`, `<footer>` — required by SEO-04 |
| `lang` attribute | `<html lang="en">` — required |

**Source:** RESEARCH.md pitfall 4 (pure black + accessibility), REQUIREMENTS.md SEO-04
